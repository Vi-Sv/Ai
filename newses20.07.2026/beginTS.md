1) заходим на лист ВВОД_CONST
2) строка 5, графа F
3) берем первое значение и уносим на новый лист в графу D, строка 2 (это наш уровень 1)
4) система фиксирует данное значение в памяти, возвращается на ВВОД_CONST, в графу F, со строки 5
5) теперь наша задача искать совпадения и забирать соответствующие данные:
6) система держит в памяти то самое значение, которое сейчас лежит на новом листе (значение уровня 1), движется по графе F и проверяет, как только нашло совпадение, задерживается на строчке, уходит в графу D, забирает значение и переносит строго ПОД значение уровня 1. Это наш уровень 2
7) Как только система соберет для уровня 1 все соответствующие себе значения из графы D (уровень 2), запускается новый цикл
8) Теперь, система движется по созданному списку из уровня 1 и 2, берет в память значение уровня 1 + первое значение уровня 2 (создается уникальная комбинация для уровня 1 и первого значения уровня 2), идет на лист ВВОД_CONST, проверяет 2 значения - значение уровня 1, и значение уровня 2, как только нашел совпадение, задерживается на строке, идет в графу H, забирает значение, и утаскивает , вставляя СТРОГО после первого значения уровня 2
9) повторяет, пока не заберет все для уникальной комбинации из значения уровня 1 и первого значения уровня 2, как только забрал все данные, проверяет следующую уникальную комбинацию - значение 1 и второе значение уровня 2, (забирает нужное значение графы H) и вставялет строго ПОД второе значением уровня 2.
10) у нас получается так: группа, во главе которого значение 1 уровня, подгруппы со значениями 2 уровня, и микро подгруппы для каждого значения 2, формируя 3 уровень
11) как только одна такая группа сформирована, система смотрит на значение уровня 1, и начинает проверять значения из листа ВВОД_COSNT на "несоответствие" (исключаем дубли), теперь, система ищет первое не соответсвующее значение, и выполняет весь тот же алгоритм по созданию гуппы и 3 уровней, 
12) таким образом нужно собрать ВСЕ возмоные группы для значений из графы F листа ВВОД_CONST
13) все операции по копировать вставить необходимо делать через "значения", или иначе - так как данные на лситах могут лежать в формулах
14) после каждой группы, создавать пустую строчку, чтобы все 3 уровня можно ыло обернуть в физическую группу (плюсик), при этом при отработке макроса, все группы по умпочланию должны быть "схлопнуты", внимание - при развертывании, групы не должны раскрываться 
15) 5 строка листа ВВОД_COSNT - это первая строка, где начинаются нужны енам данные, F - родитель D, а F+D родитель для H
16) макрос выдавать по блочно, так как чат ограничиывает вывод, дели на блоик, выдавай, подтрвеждай вывод блока и запрашивай новый
