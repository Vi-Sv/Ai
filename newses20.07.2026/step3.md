## Полное техническое задание на разработку макроса VBA (Excel)
Цель: Создание иерархического структурированного справочника с 3 уровнями вложенности, сквозной нумерацией и специфическим визуальным форматированием в новой книге.
------------------------------
## 1. Исходные данные и структура источника

* Название исходного листа: ВВОД_CONST в текущей рабочей книге.
* Границы данных: Данные начинаются со строки 5 (все строки выше полностью игнорируются).
* Связи и иерархия данных (Родитель -> Потомок):
* Уровень 1 (Главный родитель): Строится на основе уникальных значений столбца F.
   * Уровень 2 (Подгруппа): Строится на основе уникальных значений столбца D, привязанных к значению Уровня 1 из той же строки.
   * Уровень 3 (Элемент): Строится на основе уникальных значений столбца H, привязанных к уникальной комбинации родительских значений Уровня 1 + Уровня 2 из той же строки.
* Сопутствующие данные (забираются строго со строки Уровня 3):
* Значение из столбца A исходного листа.
   * Значение из столбца L исходного листа.
   * Значение из столбца K исходного листа.
   * Значение из столбца M исходного листа.
   (Исходный столбец N полностью исключен из обработки).

------------------------------
## 2. Бизнес-логика, оптимизация и обработка в памяти

* Исключение дубликатов: Группы, подгруппы и элементы не должны дублироваться. В итоговом документе каждое значение Уровня 1 уникально. Внутри него уникально каждое значение Уровня 2. Внутри Уровня 2 уникально каждое значение Уровня 3.
* Игнорирование пустых ячеек: Любые строки с пустыми значениями в столбцах F, D или H автоматически отсекаются на этапе импорта.
* Производительность: Построчный перебор ячеек на листе, использование методов .Select/.Activate, а также прямое копирование через буфер обмена (Copy/Paste) категорически запрещены. Данные диапазона A1:M[последняя_строка] должны считываться в массив типа Variant одной операцией. Дерево связей строится в оперативной памяти с помощью вложенных объектов Scripting.Dictionary.

------------------------------
## 3. Формирование структуры на итоговом листе

* Создание файла: Все результаты выводятся в новую рабочую книгу на первый лист (Sheet1).
* Тип данных: Все данные выгружаются строго как текстовые или числовые значения (без формул).
* Строка 1 (Пустая): Оставляется полностью пустой под возможные будущие нужды.
* Строка 2 (Шапка таблицы): В ячейки записываются следующие названия заголовков:
* Ячейка C2 = "№ п.п."
   * Ячейка D2 = "Объект"
   * Ячейка E2 = "Ед. изм."
   * Ячейка F2 = "Исх. объем"
   * Ячейка G2 = "Норма на ед."
* Выгрузка иерархии (начиная со строки 3 в столбце D):
Вывод элементов в группе происходит сверху вниз в следующей хронологии:
1. Значение Уровня 1.
   2. Строго под ним — первое значение Уровня 2 (в текстовое значение в обязательном порядке добавляются 4 пробела в начале строки для создания визуального отступа).
   3. Строго под ним — поочередно все относящиеся элементы Уровня 3.
   4. Сопутствующие данные для элементов Уровня 3 распределяются по строке:
   * Из исходного столбца A -> в столбец A нового листа.
      * Из исходного столбца L -> в столбец E нового листа.
      * Из исходного столбца K -> в столбец F нового листа.
      * Из исходного столбца M -> в столбец G нового листа.
   5. Переход к следующему значению Уровня 2 (с 4 пробелами в начале) и его элементам Уровня 3.
* Разделители групп: После полной выгрузки всей ветки Уровня 1 (включая все подгруппы 2 и 3 уровней) макрос обязан вставить одну полностью пустую строку перед выводом следующей родительской группы Уровня 1.

------------------------------
## 4. Правила сквозной нумерации (Столбец C)
Перед заполнением столбца C ему принудительно присваивается текстовый формат ячеек (NumberFormat = "@"). Для каждой выгружаемой строки формируется хронологический номер:

* Для Уровня 1: Простой порядковый номер группы (1, 2, 3, ...).
* Для Уровня 2: Номер формата Родитель.Подгруппа (1.1, 1.2, 2.1, ...).
* Для Уровня 3: Номер формата Родитель.Подгруппа.Элемент (1.1.1, 1.1.2, 1.2.1, ...).

------------------------------
## 5. Визуальное оформление и форматирование ячеек

* Шрифт: Абсолютно для всех заполненных ячеек таблицы (в диапазоне от столбца A до G, начиная со строки 2 до конца данных) устанавливается жирный шрифт (Font.Bold = True).
* Оформление Уровня 1: Высота строки устанавливается в 30. Диапазон ячеек строки от столбца C до G заливается темно-серым цветом (RGB 58, 58, 58). Цвет шрифта устанавливается белым (RGB 255, 255, 255).
* Оформление Уровня 2: Диапазон ячеек строки от столбца C до G заливается более светлым серым цветом (RGB 122, 122, 122). Цвет шрифта устанавливается белым (RGB 255, 255, 255).
* Параметры столбцов и выравнивание:
* Строка 2 (Шапка): Высота 25. Выравнивание по центру (по вертикали и горизонтали), включен перенос текста.
   * Столбец А: Ширина 10. Выравнивание строго по центру (по вертикали и горизонтали), включен перенос текста.
   * Столбец C: Выравнивание строго по центру (по вертикали и горизонтали).
   * Столбец D: Ширина 48. Выравнивание стандартное, включен перенос текста.
   * Столбец E: Выравнивание строго по центру (по вертикали и горизонтали).
   * Столбец F и G: Ширина 9. Выравнивание строго по центру (по вертикали и горизонтали). Устанавливается числовой формат с двумя знаками после запятой (NumberFormat = "0.00").

------------------------------
## 6. Физическая группировка строк (Структура / Outline)

* Положение кнопок: Элементы управления группировкой (плюсы) настраиваются для отображения над данными (ws.Outline.SummaryRow = xlSummaryAbove).
* Внутренняя группировка: Строки со значениями Уровня 3 физически группируются под своей строкой Уровня 2.
* Внешняя группировка: Строки со значениями Уровней 2 и 3 физически группируются под строкой своего Уровня 1. Границы определяются пустыми строками-разделителями.
* Поведение при раскрытии (Изоляция уровней):
* По умолчанию вся структура на листе полностью сворачивается до базового отображения Уровня 1.
   * Логика изоляции: При клике пользователем на значок [+] строки Уровня 1 должны отобразиться только вложенные в него строки Уровня 2. Строки Уровня 3 внутри этих подгрупп обязаны оставаться скрытыми (свернутыми) до тех пор, пока пользователь вручную не нажмет на [+] конкретной подгруппы Уровня 2. Одновременное сквозное раскрытие всех подуровней при открытии первого уровня запрещено.
